# Sprint 4 - Project Content Model

## Goal
Define structured case study template.

## Content Fields (required per project)
- title
- role
- timeline
- tools
- problem
- constraints
- hypothesis
- process
- solution
- metrics
- evidence_links
- lessons

## Tasks
- Create reusable HTML structure for case study
- Enforce section order
- Add placeholder content

## Files
- projects/template.html

## Acceptance Criteria
- All required fields present
- Every claim paired with metric or artifact
- No empty sections allowed

## Verification Steps
- Manually check each section exists
- Confirm at least one "receipt" per major claim

## What NOT to Do
- No storytelling fluff
- No vague claims without metric
